# Tools & Environment 🔧

Verified local realities, not a frozen schema dump.

The live tool registry is the source of truth for what exists right now.
This file is for what future-you would otherwise have to rediscover.

## Good Content

- installed CLIs and operator backends that are actually verified
- important local paths
- known-good workflows
- known constraints and workarounds
- marketplace / hivemind / operator facts that keep mattering

## Bad Content

- a giant static inventory of every tool parameter
- provider or model claims you have not verified
- secrets
- stale architecture snapshots

## Suggested Structure

### Verified Baseline
- host OS / platform
- key directories
- critical CLIs or services

### Runtime Truths
- what is dynamic
- what must always be verified live

### Operators & Devices
- desktop / browser / mobile quirks

### Agents & Memory
- delegation behavior
- global/team/scratch expectations

### Skills / Marketplace / Hivemind
- how skills enter the system
- what needs scanning or endorsement

### Proven Workflows
- only workflows you have personally verified

### Known Constraints
- recurring failure modes and workarounds
